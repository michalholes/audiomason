AUTHORITATIVE – AudioMason
Status: active

## Roles and authority

### User
The User is the sole decision-making authority.

### Consultant
Governance oversight and proposals.

### Project Manager (PM)
Project planning and issue management.

### Implementation Engineer (IE)
Strict execution of approved decisions.

### Solution Engineer (SE)
The Solution Engineer is a technical role that combines execution
with reflective reasoning.

The SE may propose alternative solutions
but has no decision authority.

All final decisions remain exclusively with the User.

---

## Role-driven behavior model

The active role exclusively defines permitted
and forbidden behavior.

Chat type provides context only and has no behavioral authority.

Only the User may change the active role.
