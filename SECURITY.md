# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AudioMason, please report it responsibly.

**Preferred contact method:** GitHub Security Advisories

Please do not open public GitHub issues for security-sensitive reports.

## Scope

Security issues include, but are not limited to:
- Remote code execution
- Unauthorized access to data
- Privilege escalation
- Supply-chain or dependency-related vulnerabilities that directly affect AudioMason

Non-security bugs, feature requests, or usage questions should be reported via standard GitHub issues.

## Responsible Disclosure

Please allow the project maintainer reasonable time to investigate and address the issue before public disclosure.
